# PRD: 트렌드 코리아 (대한민국 사건/이슈 타임라인 서비스)

> 작성일: 2026.02.13
> 기반 문서: 트코 기획서 v1.1

---

## 1. 제품 정의

### 1.1 한 줄 정의

대한민국에서 발생하는 사건, 법률 개정, 사회적 특이점을 시간순으로 기록하고, 이슈가 된 사건을 지속 추적하여 제공하는 정보 아카이브 서비스

### 1.2 핵심 가치

| 가치        | 설명                                        | 측정 지표             |
| ----------- | ------------------------------------------- | --------------------- |
| 기록        | 일자별로 정리된 신뢰 가능한 사건 아카이브   | 일평균 등록 사건 수   |
| 추적        | 이슈의 상태/업데이트 로그 기반 최신화       | 이슈당 평균 트리거 수 |
| 참여        | 커뮤니티를 통한 맥락 공유 및 관심 이슈 확산 | DAU, 게시글/댓글 수   |
| 한눈에 보기 | 메인에서 속보/트렌드/지표를 빠르게 파악     | 메인 페이지 체류 시간 |

### 1.3 타겟 사용자

| 페르소나    | 설명                                          | 핵심 니즈                               |
| ----------- | --------------------------------------------- | --------------------------------------- |
| 뉴스 소비자 | 매일 시사 이슈를 챙기는 2030 직장인           | 흩어진 뉴스를 한 곳에서 시간순으로 파악 |
| 이슈 추적자 | 특정 사건의 경과를 지속적으로 추적하는 사용자 | 관심 이슈의 상태 변화 알림              |
| 토론 참여자 | 사건/이슈에 대해 의견을 나누고 싶은 사용자    | 맥락 있는 커뮤니티 토론                 |

## 2. 도메인 모델

### 2.1 핵심 엔티티

```
Event (사건)
├── id: string
├── occurredAt: DateTime        # 발생 일시
├── title: string               # 제목 (최대 50자)
├── summary: string             # 2~3줄 요약
├── tags: Tag[]                 # 분야/지역 태그 (최대 3개)
├── sources: Source[]           # 출처 목록
├── importance: low | medium | high
├── verificationStatus: verified | unverified
└── relatedIssueIds: string[]   # 연결된 이슈

Issue (이슈)
├── id: string
├── title: string               # 제목 (최대 50자)
├── description: string         # 배경 및 요약
├── status: ongoing | closed | reignited | unverified
├── tags: Tag[]
├── triggers: Trigger[]         # 업데이트 타임라인
├── trackerCount: number        # 추적자 수
├── relatedEventIds: string[]   # 관련 사건
└── sources: Source[]

Trigger (트리거 / 이슈 업데이트)
├── id: string
├── issueId: string
├── occurredAt: DateTime
├── summary: string             # 변경 내용 요약
├── type: article | ruling | announcement | correction | status_change
└── sources: Source[]

Post (커뮤니티 게시글)
├── id: string
├── authorId: string
├── title: string               # 최대 100자
├── content: string             # 마크다운 본문
├── tags: Tag[]                 # 최대 3개
├── isAnonymous: boolean
├── likeCount: number
├── dislikeCount: number
├── commentCount: number
└── createdAt: DateTime

Comment (댓글)
├── id: string
├── postId: string
├── parentId: string | null     # 대댓글 시 부모 ID
├── authorId: string
├── content: string
├── likeCount: number
└── createdAt: DateTime

User (사용자)
├── id: string
├── nickname: string
├── email: string
├── profileImage: string | null
├── role: guest | member | admin
├── socialProviders: SocialProvider[]  # kakao | naver | google
├── trackedIssueIds: string[]
├── savedEventIds: string[]
└── createdAt: DateTime

Tag (태그)
├── id: string
├── name: string                # 예: 정치, 사법, 재난
├── type: category | region     # 분야 또는 지역
└── slug: string

Source (출처)
├── url: string
├── title: string
├── publisher: string
└── publishedAt: DateTime
```

### 2.2 상태 정의

**이슈 상태 (Issue Status)**

| 상태     | 코드         | 색상   | 설명                |
| -------- | ------------ | ------ | ------------------- |
| 진행중   | `ongoing`    | red    | 현재 진행 중인 이슈 |
| 종결     | `closed`     | green  | 마무리된 이슈       |
| 재점화   | `reignited`  | yellow | 종결 후 재부상      |
| 확인필요 | `unverified` | gray   | 사실 확인 필요      |

**사건 검증 상태 (Verification Status)**

| 상태     | 코드         | 설명                  |
| -------- | ------------ | --------------------- |
| 확인됨   | `verified`   | 출처로 사실 확인 완료 |
| 확인필요 | `unverified` | 확인 대기             |

## 3. 페이지 및 기능 요구사항

### 3.1 정보 구조 (IA)

```
/                          # 메인 (홈)
/timeline                  # 타임라인
/timeline/:date            # 특정 날짜 타임라인
/issues                    # 이슈 추적 목록
/issues/:id                # 이슈 상세
/community                 # 커뮤니티
/community/write           # 글쓰기
/community/:id             # 게시글 상세
/search                    # 검색
/search?q=                 # 검색 결과
/tracking                  # 내 추적
/mypage                    # MY 페이지
/mypage/edit               # 회원 정보 수정
```

### 3.2 메인 페이지 (`/`)

허브 역할로, 각 섹션의 최신 정보를 요약 제공

| 섹션               | 데이터                                       | MVP     |
| ------------------ | -------------------------------------------- | ------- |
| 타임라인 미니맵    | 최근 7일 사건 밀도 시각화, 날짜 클릭시 이동  | O       |
| 속보               | 최신 사건/트리거 10건 (시각, 제목, 1줄 요약) | O       |
| 커뮤니티 핫 게시물 | 최근 24시간 인기 게시물 5건                  | O       |
| 실시간 검색순위    | 1~10위 키워드 + 순위 변동 (1시간 갱신)       | O       |
| 대한민국 트렌드    | 24h/7d/30d 급상승 이슈/태그 (최대 5개)       | O       |
| 지표 스트립        | 환율, 출산률, 금리 등 가로 스크롤 캐러셀     | Phase 2 |
| 입법 현황 요약     | 최근 법안 5건 (발의/심사/통과/공포)          | Phase 2 |

### 3.3 타임라인 페이지 (`/timeline`)

일자별 사건 아카이브, 무한 스크롤로 과거 탐색

**기능 요구사항:**

- FR-T1: 날짜 선택기 (달력 모달, 사건 발생일 점 표시, 오늘로 이동)
- FR-T2: 필터 (분야, 지역, 중요도)
- FR-T3: 정렬 (시간순 기본 / 중요도순)
- FR-T4: 사건 카드 (발생 시각, 제목, 요약, 태그, 출처 수, 상태 뱃지, 저장/공유)
- FR-T5: 무한 스크롤 (일주일 단위 로딩)
- FR-T6: 사건 상세 모달 (전체 내용, 출처 링크, 관련 이슈 연결)

### 3.4 이슈 추적 페이지 (`/issues`)

언론/SNS 주요 이슈의 최신 상태 추적

**기능 요구사항:**

- FR-I1: 필터 (상태, 분야, 기간)
- FR-I2: 정렬 (최신 업데이트순 기본 / 추적자순 / 중요도순)
- FR-I3: 이슈 카드 (상태 아이콘, 제목, 최근 업데이트, 최신 변경 요약, 태그, 추적자 수, 추적/상세 버튼)
- FR-I4: 이슈 상세 페이지 (개요, 업데이트 타임라인, 관련 사건, 출처)
- FR-I5: 추적 등록/해제 (로그인 필요)

### 3.5 커뮤니티 페이지 (`/community`)

사용자 간 의견 공유 및 토론

**기능 요구사항:**

- FR-C1: 탭 (최신 / 인기(24h 추천) / 핫(추천+댓글 복합))
- FR-C2: 게시글 카드 (제목, 작성자, 작성 시각, 추천/댓글 수, 태그)
- FR-C3: 글쓰기 (제목 필수, 본문 에디터, 태그 최대 3개, 익명 선택)
- FR-C4: 게시글 상세 (전체 내용, 댓글/대댓글 트리, 추천/비추천, 신고)
- FR-C5: 정렬 (최신순 기본 / 추천순 / 댓글 많은 순)

### 3.6 검색 페이지 (`/search`)

사건/이슈/게시글 통합 검색

**기능 요구사항:**

- FR-S1: 검색창 (실시간 자동완성, 최근 검색어 10개, 삭제)
- FR-S2: 고급 필터 (기간, 분야, 지역, 중요도)
- FR-S3: 결과 탭 (전체 / 사건 / 이슈 / 커뮤니티)
- FR-S4: 검색 결과 카드 (제목 하이라이트, 출처 유형, 날짜, 요약, 태그)
- FR-S5: 정렬 (관련도순 기본 / 최신순 / 인기순)

### 3.7 내 추적 페이지 (`/tracking`)

관심 등록한 사건/이슈 모아보기

**기능 요구사항:**

- FR-M1: 탭 (추적 중인 이슈 / 저장한 사건)
- FR-M2: 정렬 (최신 업데이트순 기본 / 저장 날짜순 / 이름순)
- FR-M3: 카드 (NEW 뱃지, 제목, 최근 업데이트, 상태, 상세보기/추적해제)
- FR-M4: 빈 상태 처리 ("추적 중인 항목이 없습니다" + 안내)

### 3.8 MY 페이지 (`/mypage`)

회원 정보 및 계정 관리

**기능 요구사항:**

- FR-MY1: 프로필 (이미지, 닉네임, 이메일, 가입일)
- FR-MY2: 계정 관리 (회원 정보 수정, 비밀번호 변경, SNS 연동/해제)
- FR-MY3: 서비스 정보 (공지사항, 이용약관, 개인정보처리방침, 문의)
- FR-MY4: 로그아웃 / 회원탈퇴

## 4. 공통 시스템

### 4.1 인증/인가

| 항목      | 상세                                        |
| --------- | ------------------------------------------- |
| 가입 방식 | 이메일 + 비밀번호, SNS (카카오/네이버/구글) |
| 인증      | JWT (Access Token + Refresh Token)          |
| 권한      | guest / member / admin                      |

**권한 매트릭스:**

| 기능                          | guest | member | admin |
| ----------------------------- | ----- | ------ | ----- |
| 사건/이슈/커뮤니티 조회, 검색 | O     | O      | O     |
| 사건/이슈 저장                | X     | O      | O     |
| 게시글/댓글 작성              | X     | O      | O     |
| 추천/비추천                   | X     | O      | O     |
| 사건/이슈 등록/수정           | X     | X      | O     |

### 4.2 공통 필터 체계

| 필터   | 옵션                                               | 적용 페이지               |
| ------ | -------------------------------------------------- | ------------------------- |
| 기간   | 전체 / 1주일 / 1개월 / 3개월 / 직접 입력           | 타임라인, 이슈 추적, 검색 |
| 분야   | 정치, 사회, 사법, 재난, 경제, 노동, 외교안보, 문화 | 전체                      |
| 지역   | 전국, 서울, 경기, 인천, 강원, 충북, 충남...        | 타임라인, 검색            |
| 중요도 | 전체 / 높음 / 중간 / 낮음                          | 타임라인, 검색            |
| 상태   | 진행중 / 종결 / 재점화 / 확인필요                  | 이슈 추적 전용            |

### 4.3 페이지 간 연결 구조

```
메인 ──> 타임라인 ──> 사건 상세 ──> 관련 이슈
  │                                     ↑
  ├──> 이슈 추적 ──> 이슈 상세 ──> 관련 사건
  │
  ├──> 커뮤니티 ──> 게시글 상세 ──> 관련 사건/이슈 태그
  │
  ├──> 검색 ──> 사건/이슈/커뮤니티 결과
  │
  └──> 내 추적 ──> 저장한 사건/추적 이슈 상세
```

## 7. MVP 범위

### 7.1 MVP 포함

- 메인 (타임라인 미니맵, 속보, 커뮤니티 핫, 검색순위, 트렌드)
- 타임라인 (일자별 사건 아카이브, 무한 스크롤, 필터/정렬, 상세 모달)
- 이슈 추적 (목록, 상세, 업데이트 타임라인, 추적 등록)
- 커뮤니티 (게시글 CRUD, 댓글/대댓글, 추천/비추천, 탭 필터)
- 검색 (통합 검색, 고급 필터, 탭별 결과, 자동완성)
- 내 추적 (관심 사건/이슈 모아보기, NEW 뱃지)
- MY (회원 정보, SNS 연동, 서비스 정보)
- 인증 (이메일 + SNS 로그인)

### 7.2 Phase 2

- 핫딜 (크롤링 기반 할인 정보)
- 입법 현황 (법안 추적)
- 지표 스트립 (환율, 출산률 등)
- 알림 시스템 (푸시/이메일)
- 게시글 이미지 첨부
- 다크 모드 / 테마 설정

### 7.3 장기 로드맵

- 지표 상세 (시계열 그래프)
- AI 요약
- 팔로우 (유저 구독)
- 프리미엄 멤버십
- 오픈 API
- 데이터 내보내기 (CSV/JSON)

| 출처   | Source  | `source`  | 뉴스 기사/공식 발표 등 참고 자료 |
