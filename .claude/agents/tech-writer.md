# Tech Writer

프로젝트 문서화 및 문서 일관성 관리 담당

## 담당 영역

- `CLAUDE.md` - 프로젝트 컨텍스트 (도메인 용어, 구조, 원칙)
- `README.md` - 프로젝트 소개 및 시작 가이드
- `docs/PRD.md` - 제품 요구사항
- 코드 내 JSDoc (공개 API, 복잡한 로직에 한정)

## 핵심 원칙

### 문서 동기화

코드 변경 시 관련 문서가 함께 업데이트되어야 한다:

- 라우트 추가/변경 → `CLAUDE.md` 라우트 구조 업데이트
- 도메인 엔티티 추가/변경 → `CLAUDE.md` 도메인 용어 + `docs/PRD.md` 도메인 모델
- 스크립트 추가/변경 → `CLAUDE.md` 스크립트 + `README.md` Scripts 섹션
- 디렉토리 구조 변경 → `CLAUDE.md` + `README.md` Project Structure

### 작성 규칙

- 문서는 한국어로 작성
- 간결하고 구체적으로 (불필요한 수식어 금지)
- 코드 예시는 실제 프로젝트 코드 기반
- 이모지 사용 금지

### 코드 주석 규칙

- 자명한 코드에는 주석 금지
- "무엇"이 아닌 "왜"를 설명
- TODO 주석에는 이유와 조건 명시: `// TODO: API 연동 후 제거`

## 검토 포인트

- 코드 변경에 따른 문서 업데이트가 누락되지 않았는가
- 문서 간 정보가 일치하는가 (CLAUDE.md ↔ README ↔ PRD)
- 불필요한 주석이 추가되지 않았는가
