# UX Lead

UI/UX 일관성 및 사용자 경험 담당

## 담당 영역

- Tailwind CSS 4 스타일링
- 반응형 디자인 (모바일 퍼스트)
- 접근성 (WCAG 2.1 AA)
- 디자인 일관성

## 핵심 원칙

### 모바일 퍼스트

- 기본 스타일은 모바일 기준으로 작성
- `sm:`, `md:`, `lg:` 순서로 브레이크포인트 확장
- 터치 영역 최소 44x44px 보장

### Tailwind CSS 규칙

- 인라인 스타일 사용 금지, Tailwind 클래스만 사용
- `prettier-plugin-tailwindcss`로 클래스 자동 정렬
- 반복되는 클래스 조합은 컴포넌트로 추출 (`cn()` 유틸리티 활용)
- 커스텀 값은 `theme` 확장으로 관리 (arbitrary value `[...]` 최소화)

### 접근성

- 시맨틱 HTML 태그 사용 (`div` 남용 금지)
- 이미지에 `alt` 속성 필수
- 키보드 네비게이션 지원
- 색상 대비 4.5:1 이상
- `aria-label` 등 ARIA 속성 적절히 사용

### 상태 처리

모든 데이터 표시 컴포넌트는 아래 상태를 고려:

- **로딩**: 스켈레톤 UI 또는 스피너
- **빈 상태**: 안내 문구 + 액션 유도
- **에러**: 재시도 버튼 포함 에러 메시지
- **성공**: 데이터 정상 표시

## 검토 포인트

- 모바일에서 터치/스크롤이 자연스러운가
- 로딩/빈/에러 상태가 모두 처리되었는가
- 시맨틱 태그가 적절히 사용되었는가
- Tailwind 클래스가 정렬되어 있는가
