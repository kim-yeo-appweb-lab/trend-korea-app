# 테스트 컨벤션

## 파일 위치

- 테스트 파일은 대상 파일과 같은 디렉토리에 `*.test.ts(x)` 형태로 배치
- E2E 테스트는 프로젝트 루트의 `tests/` 또는 `e2e/` 디렉토리에 배치

## 명명 규칙

- `describe`: 테스트 대상 (함수명, 컴포넌트명)
- `it` / `test`: "~해야 한다" 형태의 한국어 서술
- 예: `it("빈 입력일 때 에러 메시지를 표시해야 한다")`

## 테스트 트로피 (비중 가이드)

- **Static Analysis** (20%): TypeScript + ESLint로 자동 검출
- **Unit** (30%): 순수 함수, 유틸리티, 핵심 비즈니스 로직
- **Integration** (40%): 컴포넌트 간 연동, API 호출
- **E2E** (10%): 핵심 사용자 플로우만 선별

## 작성 원칙

- 테스트는 구현이 아닌 **동작**을 검증한다
- AAA 패턴: Arrange(준비) → Act(실행) → Assert(검증)
- 테스트 간 상태 공유 금지 (각 테스트는 독립적)
- 모킹은 외부 의존성(API, 파일시스템)에만 사용, 내부 모듈 모킹 지양
