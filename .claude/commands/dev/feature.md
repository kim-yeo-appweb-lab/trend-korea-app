ROADMAP 태스크 기반으로 기능을 개발한다.

## 수행 절차

### 1단계: 태스크 선택

1. `docs/ROADMAP.md`를 읽어 현재 마일스톤과 태스크 목록을 파악한다.
2. `[ ]` (미완료) 상태인 태스크를 사용자에게 보여주고 작업할 태스크를 선택받는다.
3. 선택된 태스크의 요구사항을 `docs/PRD.md`에서 확인한다.

### 2단계: 코드 구현

1. 관련 코드베이스를 탐색하여 현재 구조를 파악한다.
2. `.claude/rules/` 하위 규칙을 준수하며 구현한다.
3. 구현 중 아키텍처 결정이 필요하면 사용자에게 확인한다.

### 3단계: 검증

1. 타입 체크: `package.json`의 typecheck 스크립트 실행 (없으면 `npx tsc --noEmit`)
2. 린트: `package.json`의 lint 스크립트 실행 (설정되어 있는 경우)
3. 테스트: 관련 테스트가 있으면 실행, 테스트 컨벤션은 `testing.md` 참조

### 4단계: ROADMAP 업데이트

`/docs:update-roadmap` 커맨드를 실행하여 ROADMAP 상태를 최신화한다.
이 커맨드는 코드베이스를 분석하여 완료된 태스크를 자동으로 식별하고 체크박스를 업데이트한다.

## 주의사항

- 한 번에 하나의 태스크만 작업한다.
- PRD 범위를 벗어나는 구현은 사용자 확인 후 진행한다.
- 코드 파일만 수정하고, 설정 파일 변경이 필요하면 사용자에게 알린다.

## 관련 커맨드

- `/dev:review` — 구현 완료 후 코드 리뷰
- `/git:commit` — 커밋 생성
- `/docs:update-roadmap` — ROADMAP 상태 최신화
