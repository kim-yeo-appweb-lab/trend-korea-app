docs/ROADMAP.md 파일의 태스크 완료 상태를 최신화한다.

## 실행 방법

이 작업은 **Agent tool**을 사용하여 `prd-roadmap-generator` 서브에이전트에게 위임한다.

- **subagent_type**: `prd-roadmap-generator`
- **description**: "ROADMAP 상태 최신화"
- **prompt**: 이 커맨드의 "수행 절차" 이하 전체 내용을 전달한다. ROADMAP 파일 경로(`docs/ROADMAP.md`)와 프로젝트 루트 경로를 포함한다.
- **mode**: "auto"

서브에이전트가 완료되면, 결과를 사용자에게 요약하여 보고한다.

## 수행 절차

1. **현재 코드 상태 파악**: git diff 및 git log를 확인하여 최근 완료된 작업을 파악합니다.

2. **ROADMAP.md 읽기**: `docs/ROADMAP.md` 파일을 읽어 현재 태스크 체크박스 상태를 확인합니다.

3. **완료 태스크 식별**: 아래 기준으로 태스크 완료 여부를 판단합니다.
   - 해당 태스크에서 명시한 파일이 존재하고 구현이 완료되었는지 확인
   - 테스트 태스크는 테스트 결과가 기록되어 있는지 확인
   - 빌드/린트 태스크는 실제 명령어를 실행하여 통과 여부 확인

4. **ROADMAP.md 업데이트**:
   - 완료된 태스크: `- [ ]` → `- [x]`로 변경
   - 서브 체크리스트도 함께 업데이트
   - ⚠️ 경고 마크가 있는 항목 중 해결된 것은 제거
   - Phase 상태 테이블의 상태 컬럼도 일치시키기 (완료 / 진행중 / 미시작)
   - Phase 제목에 상태 표기 일치시키기

5. **정합성 검증**:
   - Phase 상태 테이블과 개별 태스크 체크박스가 일치하는지 확인
   - 모든 태스크가 완료된 Phase는 상태를 "완료"로 변경
   - 하나라도 미완료 태스크가 있는 Phase는 "진행중"으로 유지

## 주의사항

- ROADMAP.md 파일만 수정합니다. 코드 파일은 절대 수정하지 않습니다.
- 태스크 완료 여부가 불확실한 경우, 해당 태스크의 구현 파일을 직접 확인합니다.
- 변경 사항을 요약하여 사용자에게 보고합니다.
